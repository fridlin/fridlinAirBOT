## Purpose
Здесь фиксируется техническая и функциональная суть FridlinAir:
что это за бот, как он работает, что умеет и на чём основан.

## Core
FridlinAir — это Telegram-бот, который получает координаты
пользователя, запрашивает погодные модели из внешних API
и формирует локальный прогноз по малому радиусу вокруг точки.

Бот работает через Node.js и Telegram Bot API.
Данные собираются из нескольких источников и сводятся
в общий ответ с коротким временным шагом.

Основная задача — рассчитать погоду для “точки здесь и сейчас”
и ближайшей зоны, а также для выбранных точек маршрута.

Функции бота:
– получение GPS координат
– запрос погодных API
– расчёт мини-радиуса вокруг пользователя
– прогноз по коротким интервалам
– сбор и интерпретация параметров (темп., ветер, дождь)
– выдача короткого ответа в Telegram

Сильные стороны:
– простота интерфейса (бот в Telegram)
– работает на любой платформе
– быстрые ответы
– высокая точность для малой зоны
– можно расширять любыми API

Слабые места:
– внешние API могут давать разное качество данных
– ограничение погодных моделей по минутам
– интерполяцию иногда нужно додумывать самому
– зависимость от скорости ответа серверов

Технологии:
– Node.js
– Telegram Bot API
– Axios для запросов
– Open-Meteo (база)
– дополнительные источники (Apple Weather / Meteoblue — позже)
