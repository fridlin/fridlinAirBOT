02-12-25 v0.1 — Решение по сравнению со вчерашней погодой

FridlinAir НЕ хранит "вчера" в файлах, БД или отдельных JSON.
Для каждого запроса используется динамический запрос к API:

– текущая погода по точке (сейчас)
– погода 24 часа назад по этой же точке

API (например Open-Meteo) позволяют запрашивать исторические данные
в пределах последних суток по любым координатам.

Это позволяет:
• сравнивать погоду именно в той же геолокации
• не хранить данные пользователей
• не создавать отдельные файлы на города/пользователей
• избегать ошибок, когда вчера в другом городе была другая погода
• минимизировать нагрузку на систему

Это постоянное архитектурное правило для FridlinAir.
